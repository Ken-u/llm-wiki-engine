Project Layout

Project Disk Layout

这份文档解释一个项目在磁盘上会长成什么样，以及这些文件/目录分别被哪些模块消费。

1. 创建项目时初始化的目录

<projects_dir>/<project_id>/
  purpose.md
  wiki/
    index.md
    overview.md
    log.md
    entities/
    concepts/
    sources/
    queries/
    comparisons/
    synthesis/
  raw/
    sources/
  .llm-wiki/
    project.json
    ingest-cache.json
    ingest-queue.json
    chats/
    checkpoints/
    lancedb/

其中一部分目录是创建时直接建好的，一部分例如 checkpoints/、lancedb/、chats/ 会在真正用到时懒创建。

2. 各目录的职责

  • purpose.md：项目知识库用途说明，普通 chat 和 Agent 都可能读它。
  • wiki/：编译后的知识库主内容，是图谱、检索、普通 chat 的核心输入。
  • raw/sources/：用户上传的原始文档，ingest 输入，也是 Agent 的 raw 工具读取范围。
  • .llm-wiki/project.json：项目本地元数据。
  • .llm-wiki/ingest-cache.json：文档内容 hash 缓存。
  • .llm-wiki/chats/：普通 chat 的单会话消息文件。
  • .llm-wiki/conversations.json：普通 chat 会话列表。
  • .llm-wiki/checkpoints/：ingest 中间断点。
  • .llm-wiki/lancedb/：每个项目自己的向量表。

3. 哪些模块会动这些目录

  • documents/ 写 raw/sources/
  • ingest/ 读 raw、写 wiki、写 cache/checkpoint、触发 embedding
  • search/ 读 wiki 和 .llm-wiki/lancedb
  • chat/ 读写 .llm-wiki/chats 与 conversations.json
  • agents/ 读 wiki、raw、ticket wiki，但不写聊天历史到项目磁盘
  • feedback/ 根据任务状态回写 wiki 页面
