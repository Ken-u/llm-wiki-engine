Config Layers

Config And Admin Settings

这份文档解释 engine 配置是如何被加载和覆盖的，以及 admin 页面到底改动了哪一层配置。

1. 配置优先级

  1. config.yaml
  2. 环境变量
  3. 数据库 system_settings 表

后者覆盖前者，数据库层优先级最高。

2. 配置分组

  • server：host、port、projects_dir
  • auth：JWT、注册开关
  • llm：主 LLM provider、model、temperature、context size
  • embedding：embedding provider、model、dimensions
  • search：RRF 和若干打分 bonus
  • feedback：evaluator/compiler 模型配置、revision 限制、去重窗口

3. Admin 页面改的不是 YAML 文件

管理员在 UI 里修改设置时，最终是写入数据库里的 system_settings 表，然后刷新内存里的配置对象。也就是说：

  • 它不会回写原始 config.yaml
  • 重启后仍然有效，因为会再次从 DB 加载覆盖
  • 这层特别适合线上调 LLM/embedding/feedback 模型参数

4. Feedback 模型是独立配置的

feedback 下有两套专门模型配置：

  • feedback.evaluator
  • feedback.compiler

如果这些字段为空，就回退继承主 LLM；如果单独配置了 model/provider/key，就走独立链路。
