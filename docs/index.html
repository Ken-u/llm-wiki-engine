LLM Wiki Engine Docs

维护者手册索引

这里整理的是当前 llm-wiki-engine 里值得长期维护的内部实现文档。重点不是 API 用法，而是帮助维护者快速理解系统的运行链路、项目目录、关键数据结构、检索与 Agent 机制，以及反馈修复闭环。

推荐阅读顺序

  • 第一次接手 engine：先看 Engine Runtime Overview，再看 Project Disk Layout。
  • 想理解问答链路：再看 Chat And Agent Runtime。
  • 想理解数据从哪来：看 Ingest Compile Pipeline、Search And Retrieval、Knowledge Graph Generation。
  • 想理解 Agent 和反馈闭环：看 Agent Tool Runtime、Feedback Repair Loop。
  • 想改线上配置或排查模型差异：看 Config And Admin Settings。

总览与核心链路

Engine Runtime Overview 从 FastAPI 启动、项目、文档、ingest、wiki、search、chat、agent、feedback 这几个模块整体说明系统是怎么跑起来的。
Chat And Agent Runtime 解释普通项目 chat 和 Agent chat 的差异：一个是预拼上下文 RAG，一个是 toolcall orchestrator。

项目结构与数据模型

Project Disk Layout 解释项目目录里 raw/、wiki/、.llm-wiki/ 等目录和文件各自承担什么职责。
Core Data Models 解释数据库层面的核心模型：Project、Agent、AgentProject、IngestJob、FeedbackTask。

专题机制

Search And Retrieval 解释 BM25、向量检索、RRF 融合、图谱扩展与普通 chat 的 RAG 组合方式。
Agent Tool Runtime 解释 Agent 的工具集合、tool context、orchestrator 循环与调试轨迹。
Feedback Repair Loop 解释回答后异步 feedback queue、评估、重编译与人工 guidance。
Config And Admin Settings 解释配置加载优先级、管理员设置如何覆盖 YAML/环境变量，以及 feedback 模型配置。

已有专题

Ingest Compile Pipeline 详细解释当前 wiki 编译流水线。
Knowledge Graph Generation 详细解释知识图谱如何从 wiki markdown 动态生成。

维护建议

  • 改动某个模块时，优先同步更新对应专题文档，而不是只改总览页。
  • 如果一个机制已经复杂到需要在代码评审里反复解释，就值得单独补进 docs/。
  • 总览页只负责导航和阅读顺序，不承担所有实现细节。
