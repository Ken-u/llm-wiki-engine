Data Models

Core Data Models

这份文档只覆盖 engine 当前最关键的持久化模型，不展开所有用户/认证细节，重点是系统运行所依赖的对象关系。

1. Project

  • id：项目主键，同时参与磁盘目录命名。
  • slug：人类可读标识。
  • disk_path：真正的知识库磁盘目录位置。
  • ticket_project_id：可选绑定的案例库项目。
  • feedback_enabled：是否允许该项目启用 feedback 闭环。

Project 是数据库和磁盘目录之间的桥梁。

2. ProjectMember

决定用户对项目的访问与角色。当前主要角色语义是：

  • owner：完全控制，含删除对话、项目管理。
  • editor：普通可编辑成员。

多数业务 API 在真正处理项目内容前都会先做 membership 校验。

3. Agent 与 AgentProject

  • Agent：保存 agent 名称、系统提示词、是否公开、API key hash、最大工具调用次数、调试结果截断长度等。
  • AgentProject：多对多关联，定义 agent 可以访问哪些主项目。

Agent 不直接存“有哪些工具”，而是运行时根据项目上下文动态决定工具集合，例如是否暴露 ticket 工具。

4. IngestJob

  • source_path：本次编译针对哪个 raw 文件。
  • status：pending / processing / done / failed 一类状态。
  • step：0 到 3，表示队列、analysis、generation、written。
  • files_written：编译写出了哪些 wiki 文件。
  • retry_count：重试次数。

这是 ingest 队列的持久化观察点，不是知识内容本身。

5. FeedbackTask

  • 保存来源对话：conversation_id、user_message、assistant_answer
  • 保存工具轨迹：tool_traces_json、wiki_reads_json、raw_reads_json
  • 保存评估结果：target_page_path、page_exists、evaluator_confidence
  • 保存修复候选：repair_candidate_json
  • 保存人工编审状态：status、review_guidance、reject_reason、revision_count

它是 feedback 闭环的中心模型，把对话、证据、候选补丁、人工决策串在一起。
